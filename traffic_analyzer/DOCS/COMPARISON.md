# Сравнение DRAKKAR NetScanner с аналогами

## 1. Назначение сравнения

Цель сравнения — показать место DRAKKAR NetScanner среди существующих инструментов анализа сетевого трафика и объяснить, почему для дипломного проекта была выбрана собственная реализация, а не простое использование готового решения.

Для сравнения выбраны:

| Инструмент | Причина выбора для сравнения |
|---|---|
| Wireshark | Самый известный графический анализатор пакетов |
| Zeek | Сильный инструмент сетевого мониторинга и событийного анализа |
| tshark | Консольный вариант Wireshark, удобный для автоматизации |
| Fiddler Classic | Популярный HTTP/HTTPS-прокси для анализа прикладного трафика |
| DRAKKAR NetScanner | Разрабатываемая система дипломного проекта |

## 2. Краткое описание инструментов

### DRAKKAR NetScanner

DRAKKAR NetScanner — учебно-практическая система для визуального анализа сетевого трафика. Она захватывает пакеты через Scapy/Npcap, отображает их в таблице, показывает структуру выбранного пакета, строит график интенсивности и диаграмму распределения протоколов.

### Wireshark

Wireshark — профессиональный графический анализатор пакетов. Он содержит огромную базу декодеров протоколов и подходит для глубокого расследования сетевых проблем.

### Zeek

Zeek — система сетевого мониторинга, которая не просто показывает пакеты, а преобразует сетевую активность в события и журналы. Zeek часто используется в SOC и инфраструктурах мониторинга безопасности.

### tshark

tshark — консольный инструмент из семейства Wireshark. Он удобен для скриптов, серверов и автоматической обработки, но менее нагляден для начинающего пользователя.

### Fiddler

Fiddler — прокси-инструмент для анализа HTTP/HTTPS. Он силен на прикладном уровне, но не предназначен для анализа всех сетевых пакетов на уровне Ethernet/IP/TCP.

DRAKKAR NetScanner имеет низкий порог входа, потому что пользователь видит только базовые элементы: интерфейс, фильтр, старт/стоп, таблицу и графики. ## 8. Почему не использовать только Wireshark

Wireshark очень мощный, но для дипломного проекта есть причины создать собственную систему:

| Причина | Пояснение |
|---|---|
| Демонстрация разработки | Диплом требует показать собственную архитектуру и исходный код |
| Контроль над интерфейсом | Можно сделать русскоязычный интерфейс под учебную задачу |
| Контроль над логикой | Можно явно объяснить, как пакет превращается в строку таблицы и точку графика |
| Расширяемость | Проект можно дополнять собственными правилами анализа |
| Обучающий эффект | Студент показывает понимание сетевого стека, потоков и GUI |

## 9. Почему не использовать только Zeek

Zeek отлично подходит для промышленного мониторинга, но он сложнее для учебной визуальной демонстрации:

1. Он ориентирован на события и логи, а не на интерактивную таблицу пакетов.
2. Для настройки требуется понимать язык Zeek и модель событий.
3. Пользователю сложнее сразу увидеть отдельный пакет и его уровни.
4. Для красивой визуализации обычно нужны внешние панели: ELK, Grafana, Security Onion.

## 10. Почему не использовать только Fiddler

Fiddler полезен для HTTP/HTTPS, но дипломная тема связана с сетевым трафиком в целом. Fiddler не закрывает задачи:

1. Анализ ARP, ICMP, DNS, TCP без привязки к HTTP.
2. Захват низкоуровневых кадров.
3. Построение общей картины протоколов.
4. Экспорт полного PCAP-потока.

## 11. Сильные стороны DRAKKAR NetScanner

| Сильная сторона | Значение для диплома |
|---|---|
| Собственная архитектура | Можно защищать проект как инженерную разработку |
| Русский интерфейс | Удобно демонстрировать комиссии |
| Простая визуализация | Графики сразу показывают динамику трафика |
| BPF-фильтры | Пользователь изучает практический механизм фильтрации |
| PCAP-экспорт | Результат совместим с внешними инструментами |
| Модульность | Можно показать перспективы развития |

## 12. Ограничения DRAKKAR NetScanner

| Ограничение | Почему допустимо | Как можно развить |
|---|---|---|
| Меньше протоколов, чем в Wireshark | Дипломный проект не обязан повторять многолетнюю базу декодеров | Добавлять новые обработчики в `parser.py` |
| Нет сложного языка правил | Основная цель — визуализация и интерактивный анализ | Добавить модуль правил и алертов |
| Нет загрузки PCAP из файла | Главный сценарий — захват в реальном времени | Добавить импорт PCAP через Scapy |
| Нет сетевой панели мониторинга | Приложение настольное | Добавить экспорт статистики в JSON/CSV |
