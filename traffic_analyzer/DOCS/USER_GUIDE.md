# Руководство пользователя DRAKKAR NetScanner

## 1. О документе

Это руководство описывает установку, запуск и практическое использование DRAKKAR NetScanner. Документ написан подробно, чтобы его можно было использовать как часть дипломной документации и как инструкцию для демонстрации проекта.

## 2. Что делает приложение

DRAKKAR NetScanner — это настольное приложение для анализа сетевого трафика. Оно позволяет захватывать пакеты с выбранного сетевого интерфейса и сразу видеть:

1. Список пакетов в таблице.
2. Время получения каждого пакета.
3. IP-адрес или MAC-адрес источника.
4. IP-адрес или MAC-адрес назначения.
5. Протокол: TCP, UDP, DNS, HTTP, ICMP, ARP, Ethernet и другие.
6. Размер пакета.
7. Краткое описание содержимого.
8. Детальную структуру выбранного пакета.
9. График интенсивности трафика.
10. Диаграмму распределения протоколов.

## 3. Общая логика работы

```text
Пользователь выбирает интерфейс
        |
        v
При необходимости вводит BPF-фильтр
        |
        v
Нажимает "Старт"
        |
        v
Приложение захватывает пакеты
        |
        v
Таблица и графики обновляются
        |
        v
Пользователь выбирает пакет для детального анализа
        |
        v
При необходимости экспортирует результат в PCAP
```

| Компонент | Требование |

## 5. Установка на Windows

### 5.1 Установка Python

1. Скачайте Python 3.10 или новее с официального сайта Python.
2. Во время установки включите параметр добавления Python в `PATH`.
3. После установки откройте PowerShell и проверьте версию:

```powershell
python --version
```

Ожидаемый результат:

```text
Python 3.10.x
```

### 5.2 Установка Npcap

Npcap нужен Windows для захвата сетевых пакетов. Без него приложение не сможет читать трафик с сетевой карты.

Официальная ссылка:

```text
https://npcap.com/#download
```

Рекомендуемые действия:

1. Откройте ссылку.
2. Скачайте установщик Npcap.
3. Запустите установщик от имени администратора.
4. При необходимости включите режим совместимости с WinPcap.
5. Завершите установку.
6. Перезапустите PowerShell или компьютер, если интерфейсы не появились сразу.

Если Npcap не установлен, DRAKKAR NetScanner покажет окно с предложением открыть страницу загрузки.

### 5.3 Создание виртуального окружения

Перейдите в папку проекта:

```powershell
cd C:\users\Valkiria\documents\drrakarnetscanner
```

Создайте окружение:

```powershell
python -m venv .venv
```

Активируйте окружение:

```powershell
.\.venv\Scripts\Activate.ps1
```

Если PowerShell запрещает запуск скриптов, выполните:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

После активации в начале строки PowerShell появится `(.venv)`.

### 5.4 Установка зависимостей

```powershell
pip install -r traffic_analyzer\requirements.txt
```

Зависимости:

| Пакет | Назначение |
|---|---|
| PyQt6 | Графический интерфейс |
| pyqtgraph | График интенсивности трафика |
| scapy | Захват и разбор пакетов |
| psutil | Ресурсный мониторинг и расширение тестов |

### 5.5 Запуск приложения

Откройте PowerShell от имени администратора и выполните:

```powershell
cd C:\users\Valkiria\documents\drrakarnetscanner
.\.venv\Scripts\Activate.ps1
python traffic_analyzer\main.py
```

Почему нужны права администратора: драйвер захвата работает с низкоуровневым сетевым трафиком, а операционная система ограничивает такой доступ.

## 6. Установка на Linux

Установите системные зависимости:

```bash
sudo apt update
sudo apt install python3 python3-venv python3-pip libpcap-dev
```

Создайте окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r traffic_analyzer/requirements.txt
```

Запустите с правами на захват:

```bash
sudo .venv/bin/python traffic_analyzer/main.py
```

| Поле фильтра | Ввод BPF-фильтра | Позволяет снизить шум и нагрузку |

## 9. Как выбрать интерфейс

В списке интерфейсов отображаются понятные названия, например:

```text
Realtek Gaming GbE Family Controller (192.168.1.178)
Hyper-V Virtual Ethernet Adapter (192.168.160.1)
Software Loopback Interface 1 (127.0.0.1)
```

Рекомендации:

| Сценарий | Что выбрать |
|---|---|
| Анализ обычного интернет-трафика | Ethernet или Wi-Fi с вашим локальным IP |
| Анализ локальных соединений | Loopback |
| Анализ WSL/виртуальных сетей | Hyper-V или виртуальный адаптер |
| Если пакетов нет | Попробовать другой интерфейс |

## 10. BPF-фильтры

BPF-фильтр применяется до передачи пакета в Python. Это значит, что ненужные пакеты отбрасываются раньше, и приложение получает меньше данных.

| Фильтр | Что показывает | Когда использовать |
|---|---|---|
| `tcp` | Только TCP | Веб-трафик, HTTPS, большинство приложений |
| `udp` | Только UDP | DNS, игры, голосовая связь |
| `icmp` | Только ICMP | Проверка ping |
| `port 53` | Порт 53 | Анализ DNS |
| `udp port 53` | DNS поверх UDP | Чистый DNS-анализ |
| `host 8.8.8.8` | Трафик с конкретным IP | Проверка связи с узлом |
| `tcp port 443` | HTTPS на TCP | Анализ защищенных веб-соединений на транспортном уровне |
| `net 192.168.1.0/24` | Подсеть | Анализ локальной сети |

Примеры:

```text
tcp and port 443
udp port 53
host 1.1.1.1
icmp
tcp or udp
```

Если фильтр некорректен, приложение покажет ошибку и не должно зависнуть.

## 11. Таблица пакетов

Таблица содержит:

| Колонка | Значение |
|---|---|
| № | Номер пакета в текущем захвате |
| Время | Время получения пакета |
| Источник | Отправитель |
| Назначение | Получатель |
| Протокол | Определенный протокол |
| Длина | Размер пакета |
| Информация | Краткое описание |

Как читать строку:

```text
№: 15
Время: 12:31:08.214
Источник: 192.168.1.178
Назначение: 8.8.8.8
Протокол: DNS
Длина: 74
Информация: DNS запрос example.com
```

Это означает, что локальный компьютер отправил DNS-запрос к серверу.

## 12. Дерево деталей

Когда пользователь выбирает строку таблицы, в нижней левой части появляется дерево уровней пакета.

Пример:

```text
Ethernet
  src = b4:2e:99:ea:af:2d
  dst = ff:ff:ff:ff:ff:ff

IP
  src = 192.168.1.178
  dst = 8.8.8.8

UDP
  sport = 53000
  dport = 53

DNS
  qd = example.com
```

Зачем это нужно: таблица дает краткое описание, а дерево позволяет увидеть структуру пакета по уровням.

## 13. График интенсивности

График показывает количество пакетов в секунду.

```text
Пакеты/с
5 |             *
4 |         *   *       *
3 |     *   *   *   *   *
2 | *   *   *   *   *   *
1 | *   *   *   *   *   *
0 +----------------------------> секунды
    0   10  20  30  40  50  60
```

Как интерпретировать:

| Ситуация | Значение |
|---|---|
| Линия почти ровная | Трафик стабилен |
| Резкий всплеск | Открытие сайта, загрузка файла, обновление приложения |
| Долгая высокая активность | Постоянная передача данных |
| График пустой | Пакеты не поступают или выбран неверный интерфейс |

## 14. Диаграмма протоколов

Диаграмма показывает долю протоколов.

Пример:

```text
TCP      | ######################## 60%
UDP      | ########                 20%
DNS      | ####                     10%
ARP      | ##                       5%
ICMP     | ##                       5%
```

Практический смысл:

1. Много TCP обычно означает веб-трафик или работу приложений.
2. Много DNS может означать активное открытие сайтов или подозрительную доменную активность.
3. ICMP появляется при `ping`.
4. ARP характерен для локальной сети.

## 15. Экспорт PCAP

Чтобы сохранить результат:

1. Запустите захват.
2. Накопите нужные пакеты.
3. Нажмите «Стоп».
4. Нажмите «Экспорт в PCAP».
5. Выберите путь и имя файла.

PCAP можно открыть в Wireshark. Это важно для диплома: проект не изолирован, а использует стандартный формат сетевого анализа.

## 16. Рекомендуемые демонстрационные сценарии

### Сценарий 1: DNS

1. Ввести фильтр `udp port 53`.
2. Нажать «Старт».
3. В PowerShell выполнить:

```powershell
nslookup example.com
```

4. Показать DNS-пакеты в таблице.
5. Выбрать пакет и показать дерево деталей.

### Сценарий 2: ICMP

1. Ввести фильтр `icmp`.
2. Нажать «Старт».
3. Выполнить:

```powershell
ping 8.8.8.8
```

4. Показать ICMP-пакеты и график интенсивности.

### Сценарий 3: HTTPS

1. Ввести фильтр `tcp port 443`.
2. Нажать «Старт».
3. Открыть сайт в браузере.
4. Показать TCP-пакеты и рост графика.

### Сценарий 4: общий трафик

1. Оставить фильтр пустым.
2. Нажать «Старт».
3. Открыть несколько сайтов.
4. Показать диаграмму распределения протоколов.

## 17. Частые проблемы

| Проблема | Причина | Решение |
|---|---|---|
| Приложение просит Npcap | Драйвер не установлен | Скачать с `https://npcap.com/#download` |
| Нет интерфейсов | Нет Npcap или нет прав | Установить Npcap, запустить от администратора |
| Нет пакетов | Выбран не тот интерфейс | Выбрать Ethernet/Wi-Fi с локальным IP |
| Ошибка фильтра | BPF написан неверно | Использовать простой фильтр: `tcp`, `udp`, `icmp` |
| Графики пустые | Пакетов нет | Создать активность: открыть сайт, выполнить `ping` или `nslookup` |
| Экспорт пустой | Захват не выполнялся | Сначала запустить захват и накопить пакеты |

## 18. Правила безопасной работы

PCAP-файлы могут содержать:

1. IP-адреса.
2. Домены.
3. DNS-запросы.
4. HTTP-заголовки.
5. Фрагменты полезной нагрузки.

Не передавайте PCAP третьим лицам без необходимости. Для дипломной демонстрации лучше использовать тестовый трафик: `ping`, `nslookup`, открытие обычного сайта.

DRAKKAR NetScanner предоставляет понятный путь от сетевого пакета до визуального анализа. Пользователь видит не только сырые строки таблицы, но и динамику трафика, распределение протоколов и структуру выбранного пакета. Это делает приложение удобным для учебной демонстрации, лабораторной работы и защиты дипломного проекта.
