# DRAKKAR NetScanner

**DRAKKAR NetScanner** — настольное приложение для визуализации и интерактивного анализа сетевого трафика. Проект разработан как дипломная работа по теме: **«Разработка системы визуализации и интерактивного анализа сетевого трафика с применением утилиты захвата пакетов»**.

Приложение позволяет выбрать сетевой интерфейс, запустить захват пакетов, применить BPF-фильтр, просмотреть пакеты в таблице, изучить структуру выбранного пакета, увидеть график интенсивности трафика и диаграмму распределения протоколов.

## Возможности

| Возможность | Описание |
|---|---|
| Захват пакетов | Получение сетевых пакетов с выбранного интерфейса через Scapy/Npcap |
| BPF-фильтры | Отбор пакетов до обработки: `tcp`, `udp`, `icmp`, `port 53`, `host 8.8.8.8` |
| Таблица пакетов | Отображение времени, источника, назначения, протокола, длины и краткого описания |
| Дерево деталей | Просмотр уровней выбранного пакета и его полей |
| График интенсивности | Визуализация количества пакетов в секунду |
| Диаграмма протоколов | Распределение TCP, UDP, DNS, ARP, ICMP, Ethernet и других протоколов |
| Экспорт PCAP | Сохранение захваченных пакетов в стандартный PCAP-файл |
| Проверка Npcap | Предупреждение и ссылка на загрузку, если драйвер не установлен |
| Русский интерфейс | Все элементы интерфейса и документация выполнены на русском языке |

## Требования

### Windows

1. Python 3.10 или новее.
2. Npcap: <https://npcap.com/#download>.
3. PowerShell.
4. Запуск от имени администратора для реального захвата пакетов.

### Linux

1. Python 3.10 или новее.
2. `libpcap`.
3. Права `root` или разрешение на захват пакетов.

## Быстрый запуск на Windows

Откройте PowerShell в папке проекта:

```powershell
cd C:\users\Valkiria\documents\drrakarnetscanner
```

Создайте виртуальное окружение:

```powershell
python -m venv .venv
```

Активируйте окружение:

```powershell
.\.venv\Scripts\Activate.ps1
```

Установите зависимости:

```powershell
pip install -r traffic_analyzer\requirements.txt
```

Запустите приложение:

```powershell
python traffic_analyzer\main.py
```

Для захвата реального трафика PowerShell рекомендуется запускать **от имени администратора**.

Если Npcap не установлен, приложение покажет окно с предложением скачать драйвер. Также ссылка доступна в меню:

```text
Справка -> Скачать Npcap
```

## Запуск на Linux

Установите системные зависимости:

```bash
sudo apt update
sudo apt install python3 python3-venv python3-pip libpcap-dev
```

Создайте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r traffic_analyzer/requirements.txt
```

Запустите приложение:

```bash
sudo .venv/bin/python traffic_analyzer/main.py
```

## Как пользоваться

1. Запустите приложение.
2. Выберите сетевой интерфейс.
3. При необходимости введите BPF-фильтр.
4. Нажмите **Старт**.
5. Создайте сетевую активность: откройте сайт, выполните `ping` или `nslookup`.
6. Наблюдайте за таблицей и графиками.
7. Выберите строку таблицы, чтобы увидеть подробности пакета.
8. Нажмите **Стоп**, чтобы остановить захват.
9. Нажмите **Экспорт в PCAP**, если нужно сохранить результат.

## Примеры BPF-фильтров

| Фильтр | Что показывает | Пример использования |
|---|---|---|
| `tcp` | Только TCP-пакеты | Анализ веб-трафика и соединений |
| `udp` | Только UDP-пакеты | DNS, игры, голосовой трафик |
| `icmp` | ICMP-пакеты | Проверка `ping` |
| `udp port 53` | DNS-запросы и ответы | Демонстрация работы DNS |
| `tcp port 443` | HTTPS-соединения на транспортном уровне | Открытие сайтов в браузере |
| `host 8.8.8.8` | Трафик с участием конкретного IP | Проверка связи с Google DNS |
| `net 192.168.1.0/24` | Трафик подсети | Анализ локальной сети |

## Сценарий демонстрации для защиты

Рекомендуемый короткий сценарий:

1. Запустить PowerShell от имени администратора.
2. Активировать виртуальное окружение.
3. Запустить приложение:

```powershell
python traffic_analyzer\main.py
```

4. Выбрать активный Ethernet или Wi-Fi интерфейс.
5. Ввести фильтр:

```text
udp port 53
```

6. Нажать **Старт**.
7. В отдельном PowerShell выполнить:

```powershell
nslookup example.com
```

8. Показать DNS-пакеты в таблице.
9. Выбрать пакет и показать дерево деталей.
10. Показать график интенсивности и диаграмму протоколов.
11. Нажать **Стоп**.
12. Экспортировать результат в PCAP.

## Безопасность данных

PCAP-файлы могут содержать чувствительную информацию:

1. IP-адреса.
2. Домены.
3. DNS-запросы.
4. HTTP-заголовки.
5. Фрагменты полезной нагрузки.

Не передавайте PCAP-файлы третьим лицам без необходимости. Для демонстрации лучше использовать тестовые действия: `ping`, `nslookup`, открытие обычного сайта.

## Типовые проблемы

| Проблема | Причина | Решение |
|---|---|---|
| Приложение просит установить Npcap | На Windows нет драйвера захвата | Скачать Npcap: <https://npcap.com/#download> |
| Нет интерфейсов | Npcap не установлен или нет прав | Установить Npcap и запустить PowerShell от администратора |
| Пакеты не появляются | Выбран неактивный интерфейс | Выбрать Ethernet/Wi-Fi с локальным IP |
| Ошибка фильтра | Некорректный BPF | Использовать простой фильтр: `tcp`, `udp`, `icmp` |
| Графики пустые | Нет входящих пакетов | Открыть сайт, выполнить `ping` или `nslookup` |
| Экспорт пустой | Захват не был выполнен | Сначала нажать **Старт** и накопить пакеты |
